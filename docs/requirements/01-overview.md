# 01. プロジェクト概要

## 目的

イタリア旅行で訪れた／訪れたいレストランを **地域 × ジャンル** で整理して保存・共有するための Web サイト。旅程を組むときの参考にしたり、URL を共有した友人に「現地でのおすすめ／気になる店リスト」として見せたりすることを目的とする。

## 対象ユーザー

- **一次ユーザー（編集者 + 閲覧者）**: 本人。旅行前に店をリストアップし、旅行中に絞り込んで参照する。
- **二次ユーザー（閲覧者のみ）**: 本人がURLを共有した友人・知人。同じ旅行を計画している人、これからイタリアに行く人。

## スコープ

### MVP に含めるもの

- 店舗の一覧表示（カード形式）
- 地域（都市）による絞り込み
- ジャンルによる絞り込み
- 店舗の詳細表示（住所・コメント・営業情報など）
- 地図上でのピン表示
- 訪問済み／未訪問の区別

### MVP に含めないもの（将来拡張）

- ユーザー認証・複数人での共同編集
- レビュー投稿機能
- 写真の表示
- フリーワード検索（推奨機能として候補）
- ルート計画（複数店舗を結ぶ経路）
- 多言語対応

## 成功の定義

1. 旅行中、スマホでサイトを開いて「今いる都市」と「行きたいジャンル」で 3 タップ以内に候補を絞り込める
2. URL を共有された人が、ログインなしで同じリストを閲覧できる
3. 店舗の追加・編集が **旅先のスマホからでも** 完了できる（外部データソース経由）

## 対象都市（初期スコープ）

MVP リリース時点で扱う地域:

| 地域 | 補足 | 地図中心の参考座標 |
|---|---|---|
| Rome（ローマ） | Lazio 州の州都 | 41.9028, 12.4964 |
| Florence（フィレンツェ） | Toscana 州の州都 | 43.7696, 11.2558 |
| Sicily（シチリア） | 島全域。MVP では「都市」と同列の粒度で扱う | 37.6000, 14.0152 |

> 注: 「Sicily」は本来「州」レベルの地域で、ローマ・フィレンツェ（都市）と粒度が異なる。MVP では簡易的に同列の絞り込み枠として扱い、シチリア島内の店舗が増えた段階で `Palermo` / `Catania` / `Taormina` 等の都市単位に細分化する。

地図の初期表示は **絞り込まれた都市の境界に合わせた fitBounds**（実装エージェントの判断で OK）。絞り込みなしの場合は 3 地域すべてが収まる縮尺。

## 制約条件

- **公開サイト**として運用（URL 共有で誰でも閲覧可能）
- ただし検索エンジンには **インデックスさせない**（`noindex`）。詳細は [04-non-functional-req.md](04-non-functional-req.md)
- 編集者は当面 1 名（本人）
- データソースは **外部サービス**（Google Sheets を採用。詳細は [tech-stack.md](../design/tech-stack.md)）
- ホスティングは **無料枠** で完結させる
- 個人特定情報（本名、連絡先、決済情報など）はサイトに含めない
