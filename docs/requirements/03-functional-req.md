# 03. 機能要件

## 必須機能（MVP）

### F-1. 店舗一覧表示

- カード形式で店舗をグリッド表示
- カード 1 枚に表示する情報: 店名 / 都市 / ジャンル / 価格帯 / 訪問済みマーク
- 並び順: 都市 → ジャンル の順でソート（デフォルト）

### F-2. 都市による絞り込み

- 都市タグをクリック / タップして該当店舗のみを表示
- 複数選択（OR 条件）に対応
- 「全都市」表示への解除も 1 タップで可能

### F-3. ジャンルによる絞り込み

- ジャンルタグをクリック / タップして該当店舗のみを表示
- 複数選択（OR 条件）に対応
- 都市絞り込みと **AND** で組み合わせ可能（例: ローマ × ピッツェリア）

### F-4. 店舗詳細表示

- カードをクリックで詳細画面へ遷移（モーダル or 別ページ）
- 表示項目: 店名 / 住所 / 都市・エリア / ジャンル / 価格帯 / 訪問日 / 評価 / コメント / 外部 URL / タグ

### F-5. 地図ピン表示

- 一覧と並んで地図を表示（モバイルではタブ切り替え）
- 各店舗のピンを緯度経度に基づいて配置
- ピンクリックでカード or 詳細にフォーカス
- 絞り込みに連動してピンも増減する

### F-6. 訪問済みフラグの視覚化

- 訪問済みの店舗にはバッジを表示
- 一覧・地図双方で区別がつく（例: ピンの色、カードのバッジ）

### F-9. 外部データソースを開く導線

- ヘッダーに「+ 追加」ボタンを配置
- タップ／クリックで Google Sheets のシートを **別タブ** で開く（target="_blank"）
- 旅先のスマホからの追加（US-06）を最小ステップで実現
- 認証はサイト側では行わず、Google アカウント側の認証に委ねる

### F-10. 現在地表示

- 地図タブの右上に **「📍 現在地」ボタン** を配置
- 押下時のみ `navigator.geolocation.getCurrentPosition()` で位置情報を取得
- 取得成功 → 地図上に **青い丸 + 精度円** で現在地を表示し、地図中心をその位置にパン
- 取得失敗 / 拒否 → 画面下部にトーストでエラー表示（「位置情報を取得できませんでした」など）
- 2 回目以降の押下 → 地図中心を現在地に再フォーカス（再取得しない）
- 取得した座標は **クライアントのメモリ上の変数のみ** に保持し、localStorage / Cookie / サーバー送信を行わない（D-015 参照）
- HTTPS（または localhost）必須。HTTP では Geolocation API が動作しないため、本番では Vercel ドメインで自動的に満たされる
- リスト表示時は不要なため、地図タブ表示時のみ機能

### F-11. 出典情報の保持

- 各店舗に `source`（出典スラグ）と `sourceTrust`（high / medium / low）を保持
- 出典スラグの初期セット: `gambero-rosso`, `50-top-pizza`, `slow-food`, `identita-golose`, `reddit-local`, `food-blogger`, `friend`, `guidebook`, `google-maps`, `other`
- 詳細モーダルに出典名と信頼層バッジを表示
- 関連: D-016, [docs/curation-guide.md](../curation-guide.md)

### F-12. 評価メタデータ（verdict / concerns / highlights）

- AI による分析結果として、各店舗に以下を保持:
  - `verdict`: `recommended` / `neutral` / `caution` / `skip` の総合判定
  - `concerns[]`: 負のシグナル（観光客向け、割高、品質低下 等）。タイプ + severity（low / medium / high）+ オプションの note
  - `highlights[]`: 正のシグナル（家族経営、地元客多い、老舗 等）。タイプ + オプションの note
- UI 表示:
  - **カード**: verdict バッジ（🟢🟡🔴）+ 高優先度の concern / highlight を最大 2 件チップ表示
  - **詳細モーダル**: 全 concerns と highlights を note 込みで一覧表示
- 取得方法: `docs/ai-prompt-template.md` のテンプレを Claude / ChatGPT に貼り付けて結果を取得（D-018）
- 関連: D-017, D-019

### F-13. ネガティブシグナルでの除外フィルタ

- フィルタバーに **「観光客向けを除く」** などのトグルを追加
- 初期セット:
  - `観光客向けを除く`: `tourist-oriented` の severity が medium 以上の店舗を除外
  - `味の懸念ありを除く`: `mediocre-food` が medium 以上の店舗を除外
  - `skip 判定を除く`: `verdict === "skip"` を除外
- 既存のフィルタ（都市 / ジャンル / 訪問）と AND で組み合わせ
- URL クエリにも同期（`?exclude=tourist,mediocre,skip`）

### F-14. 写真の表示（D-025）

- 店舗データに **`imageUrl`** フィールド（オプション、URL 文字列）を追加
- 詳細モーダルに画像を表示（最大 480px 幅）
- 画像未指定の店舗にはダミー表示・空白でも OK
- 外部 CDN（Cloudinary / unsplash 直リンク / 食店公式画像）を想定
- D-005（写真を MVP に含めない）の方針を **Phase 5 で部分撤回**: モーダル詳細にのみ画像を表示し、一覧カードには載せない（軽量維持）

### F-15. OSM Nominatim による緯度経度自動取得（D-025）

- `pnpm geocode` で `data/restaurants.json` を走査し、`address` フィールドがあって精度の低い座標を持つ店舗を Nominatim API に問い合わせて緯度経度を更新
- Nominatim ToS 遵守:
  - リクエスト間 1 秒以上の wait（rate limit）
  - `User-Agent: italia-gohan-map/0.1` ヘッダを付与
  - 1 回の実行で最大 50 件まで
- 取得失敗時は既存値を維持（破壊的にしない）
- 差分は git diff で目視確認してから commit

## 推奨機能

### F-7. フリーワード検索

- 店名・コメント・タグを横断して検索
- インクリメンタル検索（入力に応じて絞り込み）

### F-8. 訪問済み / 未訪問のクイックトグル（排他 3 状態）

- 絞り込みバーに **排他トグル** を配置: `すべて` / `訪問済みのみ` / `未訪問のみ`
- デフォルトは `すべて`
- 都市・ジャンル絞り込みと **AND** で組み合わせ可能
