# 04. 非機能要件

## 対応端末

- **モバイル優先**（スマートフォン）。旅行中の主たる利用シーンがモバイルのため。
- PC（デスクトップ）でも閲覧可能。ただし最適化はモバイルが先。
- タブレットは PC レイアウトの縮小で対応（個別最適化はしない）。

## 対応ブラウザ

- iOS Safari（最新 2 バージョン）
- Android Chrome（最新 2 バージョン）
- PC Chrome / Edge / Safari / Firefox（最新 2 バージョン）

## パフォーマンス

- 初回読み込み完了: **2 秒以内**（4G 回線想定）
- 絞り込み操作のレスポンス: **100ms 以内**（クライアント側で処理）
- 地図描画: **1 秒以内**

## 多言語対応

- **MVP では日本語のみ**
- 店舗名はイタリア語表記をそのまま使用（例: "Trattoria Da Enzo al 29"）
- 将来的に英語・伊語対応の余地を残す（i18n フレームワーク選定は技術スタックで考慮）

## オフライン対応

- **MVP では対応しない**（旅行先のモバイルデータ通信を前提）
- ただし HTTP キャッシュは活用し、再訪問時は高速に表示
- 将来拡張で Service Worker による完全オフライン対応を検討

## 公開・運用

- **ホスティング**: 無料枠で完結（Vercel または GitHub Pages を想定）
- **独自ドメイン**: MVP では使用しない。デフォルトドメイン（`*.vercel.app` / `*.github.io`）で運用
- **費用**: 完全無料運用が大原則
- **データソース**: Google Sheets（無料、認証不要の公開シートを想定）
- **ビルド・デプロイ**: Git push に連動した自動デプロイ。データ更新は定期再ビルド（GitHub Actions など）

## SEO・公開範囲

- **検索エンジンへのインデックスは拒否**する: 全ページに `<meta name="robots" content="noindex, nofollow">` を付与し、`robots.txt` で全クローラーを Disallow
- 利用者は **URL を知っている人のみ**（本人 + 本人がシェアした友人）
- 公開ではあるが「URL シェア限定公開」の運用

## セキュリティ・プライバシー

- 公開サイトなので、店舗情報以外の **個人特定情報は含めない**
  - 含めない例: 本人の本名 / 連絡先 / 決済情報 / 同行者の名前
- データソース（Google Sheets）は **読み取り専用の公開シート** とし、API キーをクライアントに露出させない
- 編集はシート所有者のみが Google アカウントで行う

### 位置情報（F-10）の取り扱い

- **ユーザーがボタンを押したときのみ** 取得（暗黙的な常時追尾は行わない）
- 取得した座標は **クライアントメモリ上の変数のみ** に保持
- localStorage / sessionStorage / Cookie / IndexedDB に **保存しない**
- 外部サーバー / 解析サービスに **送信しない**
- ブラウザの権限ダイアログで拒否されたら、それを尊重しトーストで「許可されませんでした」と通知するのみ
- 関連: D-015

## アクセシビリティ

### 必達項目（MVP）
- セマンティックな HTML を使う（`<nav>`, `<main>`, `<article>` 等）
- カラーコントラスト: WCAG AA 相当
- キーボード操作で主要機能（絞り込み・詳細表示・地図ピンへのフォーカス）にアクセス可能
- フォーカスインジケータが視認できる
- 訪問済み / 未訪問は **色だけでなく形状・アイコン** でも区別する

### 努力目標（できれば）
- スクリーンリーダー対応（適切な `aria-label`、ライブリージョン）
- 動作減少設定（`prefers-reduced-motion`）への配慮

## 信頼性・可用性

- 個人プロジェクトのため SLA はなし
- データソース（Google Sheets）の障害時は、最後にビルドされた静的データで動作（フェイルセーフ）
